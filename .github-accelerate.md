# GitHub 加速解决方案

## 方案 1: 切换到 SSH 连接（推荐）

SSH 连接通常比 HTTPS 更稳定快速。

### 步骤：

1. **生成 SSH 密钥（如果还没有）**：
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   # 按回车使用默认路径，可以设置密码或留空
   ```

2. **添加 SSH 密钥到 GitHub**：
   ```bash
   cat ~/.ssh/id_ed25519.pub
   # 复制输出的内容
   ```
   - 访问 https://github.com/settings/keys
   - 点击 "New SSH key"
   - 粘贴公钥内容并保存

3. **切换到 SSH URL**：
   ```bash
   git remote set-url origin git@github.com:Shining2020/charm-fluence.git
   ```

## 方案 2: 使用 Git 代理（如果你有代理服务器）

### 配置 HTTP/HTTPS 代理：

```bash
# 设置全局代理（适用于所有仓库）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 或者只针对 GitHub
git config --global http.https://github.com.proxy http://127.0.0.1:7890

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

**注意**：请将 `7890` 替换为你本地代理的端口号（常见的有 7890, 10809, 8118 等）

## 方案 3: 使用 GitHub 镜像加速（通过修改 hosts）

### macOS 系统：

```bash
# 编辑 hosts 文件
sudo nano /etc/hosts

# 添加以下内容：
140.82.112.3 github.com
140.82.112.4 github.com
```

### 或者使用 GitHub 镜像服务：
```bash
# 使用 GitHub 镜像（仅适用于 clone，不适合 push）
git clone https://ghproxy.com/https://github.com/Shining2020/charm-fluence.git
```

## 方案 4: 优化 Git 配置（增加超时和缓冲区）

```bash
# 增加 HTTP 缓冲区大小
git config --global http.postBuffer 524288000

# 增加超时时间
git config --global http.lowSpeedLimit 0
git config --global http.lowSpeedTime 999999

# 使用 HTTP/1.1（更稳定）
git config --global http.version HTTP/1.1
```

## 方案 5: 使用 GitHub CLI（gh）

安装 GitHub CLI 后，可以更高效地操作：
```bash
# 安装
brew install gh

# 登录
gh auth login

# 然后使用 git 操作会自动使用优化的连接
```

## 快速测试连接速度

```bash
# 测试 GitHub 连接
time git ls-remote https://github.com/Shining2020/charm-fluence.git

# 如果使用 SSH
time git ls-remote git@github.com:Shining2020/charm-fluence.git
```

## 推荐组合方案

**最佳实践**：
1. 首选：切换到 SSH 连接（方案1）
2. 如果有代理：配置 Git 代理（方案2）
3. 同时应用：优化 Git 配置（方案4）

这样可以最大化提升 GitHub 访问速度！
